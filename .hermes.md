# ApexAurum Hermes Edition — Project Context

Self-hosted multi-agent AI platform. Forked from ApexAurum Cloud, stripped of commercial gates. 110 tools, 4 AI personalities, 3D village, Hermes bridge + MCP.

## Stack
- **Backend:** FastAPI (Python 3.11+), PostgreSQL + pgvector, SQLAlchemy
- **Frontend:** Vue 3 + Vite, Three.js/WebGL village
- **Auth:** Local mode bypass (no login required locally)
- **LLM Router:** Anthropic / OpenAI / Ollama / vLLM
- **Memory:** Neo-Cortex vector store + Dream Engine consolidation

## Key Paths
- Backend: `backend/app/`
- Frontend: `frontend/`
- Tool registry: `backend/app/tools/`
- Services: `backend/app/services/`
- Bridge clients: `bridge_clients/`

## Services of Note
- `tool_executor.py`: 110-tool registry dispatch
- `bridge_inference.py`: Hermes agent bridge (HTTP, `X-Hermes-Key`)
- `mcp/`: MCP server implementation
- `neural_memory.py`: Neo-Cortex integration
- `suno.py` / `midi.py`: Music generation pipeline
- `village_events.py`: 3D world event bus
- `llm_provider.py`: Provider routing

## Development
- Start PostgreSQL: `docker run -d --name apex-postgres -e POSTGRES_DB=apex -e POSTGRES_USER=apex -e POSTGRES_PASSWORD=apex -p 5432:5432 pgvector/pgvector:pg16`
- Backend: `cd backend && source .venv/bin/activate && uvicorn app.main:app --reload`
- Frontend: `cd frontend && npm install && npm run dev`
- UI at `http://localhost:5173`, API docs at `http://localhost:8000/docs`

## Hermes Bridge
Any agent making HTTP requests can use the full toolset:
- `GET /api/v1/hermes/tools` — list all
- `POST /api/v1/hermes/execute` — execute tool (header: `X-Hermes-Key`)

## MCP Server
Compatible with Claude Desktop, Cline, etc. Config points to backend MCP endpoint.
